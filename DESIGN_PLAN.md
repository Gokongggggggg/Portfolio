# Portfolio Design Plan

## Core Concept
Wisely's portfolio is redesigned as a security lab operating system: a dark, focused workspace where solved challenges, activity, projects, and pentest learning feel like one connected body of work.

## Product Pillars
- Solves Archive: keep HTB solves, challenge logs, activity map, search, filters, and pagination as the main proof-of-work feature.
- Lab Dashboard: home page acts as a command center, not a marketing page.
- Project Boards: projects and pentest pages become compact boards for future writeups and case studies.
- Draft-Ready Copy: page text stays lorem ipsum while the structure is ready for final writing.

## Visual Direction
- Style: dark bento grid with subtle HUD details.
- Mood: technical, premium, calm, readable.
- Avoid: heavy neon, decorative clutter, generic landing-page hero sections.
- Palette: near-black surfaces, green activity accent, blue focus accent, amber status accent.
- Typography: Space Grotesk for headings, Manrope for body, JetBrains Mono for data and console labels.

## Page Concepts
- Home: asymmetric hero plus console panel, then bento navigation into Solves, Projects, Pentest, and workflow.
- Solves: archive command center with dashboard counters, search/filter toolbar, activity heatmap, signal note, and paginated solve log.
- Projects: board layout for future project case studies.
- Pentest: board layout for recon, validation, reporting, and future methodology notes.

## UX Rules
- Keep navigation consistent on every page.
- Preserve keyboard access with skip link and visible focus states.
- Use hover and reveal motion subtly, and respect reduced-motion preferences.
- Avoid horizontal overflow on mobile.
- Keep cards useful as repeated information units, not decoration.
