Summary

Source halaman dan tampilan browser menunjukkan teks yang berbeda. Setelah dicek, tidak ada logic decoding di JavaScript. Penyebabnya adalah font custom yang disamarkan sebagai NotoSans-Regular.ttf, padahal metadata aslinya Emoji To AZ Regular.

Reading The Source

Langkah awal adalah membuka source dan DevTools. Dari sini kelihatan challenge menggunakan visualizer lyric, dan data lyric dikirim ke browser sebagai string. Karena payload sudah ada di client, fokusnya pindah ke bagaimana browser menampilkan teks tersebut.

Dynamic Analysis

Di inspector, teks HTML di sisi kanan dan tampilan user di sisi kiri tidak sama. Ini tanda kuat bahwa transformasinya bukan sekadar string biasa. Kalau JavaScript tidak melakukan decode, kandidat berikutnya adalah CSS/font rendering.

Finding The Payload

Full lyric bisa dicari dari data yang sudah ada di browser. Bagian yang menarik adalah beberapa cue terakhir yang berisi emoji. Saat bagian emoji muncul, rendered text mulai terlihat seperti format flag.

Custom Font

CSS memuat font lokal bernama NotoSans-Regular.ttf. Nama file ini misleading: setelah font dibaca, metadata aslinya adalah Emoji To AZ Regular. CMAP font tersebut memetakan emoji ke glyph name yang readable seperti v, one, braceleft, dan underscore.

Contoh mapping dari font:

  • U+1F600v
  • U+1F603one
  • U+1F601braceleft
  • U+1F60Dbraceright

Tooling

Karena case seperti ini cukup repetitive, aku bikin tool kecil untuk upload font, paste text, lalu langsung melihat output dan mapping CMAP. Tool ini berguna kalau ketemu CTF challenge yang encode pesan lewat custom font.

Simple web tool: ctf-font-decode.
